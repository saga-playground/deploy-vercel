📚 技能加载 Skills

按需加载的知识文档。模型通过 load_skill 工具拉取到上下文中。

  • ✓ research.md — 研究流程指导
  • ✓ code_generation.md — 代码生成最佳实践

💡 技能 vs 指令: 指令(instructions)每轮都生效; 技能(skills)仅在相关时按需加载，避免上下文膨胀。

// 技能加载流程 1. Eve 扫描 agent/skills/ 目录 2. 描述(description)暴露给模型作为路由提示 3. 模型判断相关时调用 load_skill 4. 技能内容追加到当前 turn 的上下文

🤝 子代理 Subagents

专项子任务委托 — 独立 instructions、tools、sandbox。

  • ✓ researcher — 深度研究子代理
  • ✓ 内置 agent 工具 — 父代理的副本
// agent/subagents/researcher/agent.ts export default defineAgent({ model: "anthropic/claude-sonnet-4-6", description: "A specialist for deep research tasks.", }); // 隔离边界: // - 声明的子代理不继承父代理的 tools/instructions // - 内置 agent 工具继承父代理全部能力

🔌 连接 Connections

MCP Server 和 OpenAPI 集成。工具名: connection__<name>__<tool>

// agent/connections/linear.ts import { defineMcpClientConnection } from "eve/connections"; export default defineMcpClientConnection({ url: "https://mcp.linear.app/sse", description: "Linear workspace", auth: { getToken: async () => ({ token: process.env.LINEAR_API_TOKEN! }), }, tools: { allow: ["search_issues"] }, });

📦 沙箱 Sandbox

隔离的 bash 执行环境，/workspace 根路径。

  • ✓ bash — 执行 shell 命令
  • ✓ read_file / write_file — 文件操作
  • ✓ glob — 模式匹配文件发现
  • ✓ grep — 搜索文件内容
// 在工具中访问沙箱 async execute({ script }, ctx) { const sandbox = await ctx.getSandbox(); await sandbox.writeTextFile({ path: "run.py", content: script }); const result = await sandbox.run({ command: "python run.py" }); return { stdout: result.stdout }; }

📡 通道 Channels

平台适配器 — 同一个 agent 逻辑服务不同平台。

  • ✓ Eve HTTP — 默认 session API
  • ✓ Slack / Discord / Teams
  • ✓ 自定义通道 — defineChannel
// agent/channels/intake.ts import { defineChannel, POST } from "eve/channels"; export default defineChannel({ routes: [ POST("/webhook", async (req, { send }) => { const result = await send(message, { auth, continuationToken }); return Response.json(result); }) ], });

🌐 HTTP API Session

Eve 核心 HTTP 接口 — 创建/恢复 session + NDJSON 流式响应。

// 1. 创建 session POST /eve/v1/session { "message": "Hello" } // 2. 流式获取响应 GET /eve/v1/session/:id/stream // 3. 多轮对话 (带 continuationToken) POST /eve/v1/session/:id { "continuationToken": "...", "message": "Follow up" } // 响应格式: NDJSON 事件流 {"type":"message.start", ...} {"type":"message.delta", "content": "..."} {"type":"tool.start", "name": "get_weather", ...} {"type":"tool.end", "output": {...}} {"type":"message.end", "continuationToken": "..."}

📜 指令 Instructions

永久系统提示词 — 每一轮对话都生效的规则和人设。

// agent/instructions.md — 最简形式 You are Eve Demo Agent — a helpful assistant. // 也支持: // - instructions.ts — 动态构建 (defineInstructions) // - instructions/ 目录 — 多文件按字母序拼接

🔄 Session 持久化 Durable

Eve 基于 Workflow SDK 构建持久化 session — 可恢复、崩溃安全、跨多轮。

核心概念

  • ✓ Session 可跨多轮对话保持状态
  • ✓ 崩溃恢复 — 中断后自动恢复
  • ✓ continuationToken 维护对话连续性
  • ✓ 可暂停等待人工输入

🎓 学习路线

推荐按以下顺序学习 Eve 各功能模块：

  1. agent.ts + instructions.md — 最小可运行项目
  2. tools/ — 定义工具 (defineTool + Zod schema)
  3. HTTP API — 创建 session, 流式响应, 多轮对话
  4. skills/ — 按需加载文档，避免上下文膨胀
  5. subagents/ — 任务委托和隔离边界
  6. connections/ — MCP/OpenAPI 外部服务集成
  7. schedules/ — Cron 定时自动化
  8. channels/ — 多平台适配
  9. sandbox/ — 隔离执行环境
