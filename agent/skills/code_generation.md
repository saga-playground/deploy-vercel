# Code Generation Procedure

When generating or refactoring code:

1. **Understand the requirements** — Ask clarifying questions if needed.
2. **Choose the right language/framework** — Match the project's existing stack.
3. **Write clean code** — Follow naming conventions, keep functions small.
4. **Handle errors** — Add appropriate error handling for edge cases.
5. **Test** — Suggest test cases or write unit tests.
6. **Document** — Add minimal necessary comments for complex logic.

## Best Practices
- DRY (Don't Repeat Yourself)
- KISS (Keep It Simple, Stupid)
- SOLID principles where applicable
- Prefer composition over inheritance
